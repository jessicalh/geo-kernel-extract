# h5-reader directory — rules

**DO NOT modify any files outside of `h5-reader/`.**

This is a standalone Qt6/VTK reference implementation. It consumes the
frozen `fileformat/analysis_file.cpp` as a source include and the frozen
HighFive headers; otherwise it is independent. It never links the
`nmr_shielding` library, never writes H5 files, never triggers
extraction.

---

## What you own

- `h5-reader/src/` — all reader source files (Qt/VTK C++).
- `h5-reader/CMakeLists.txt` + `CMakePresets.json` — build.
- `h5-reader/CLAUDE.md` — this file.
- `h5-reader/README.md` — build/run instructions for humans.
- `h5-reader/udp_listen.py` — UDP log listener (port 9997).
- `h5-reader/launch_reader.sh` — Linux/Mac launcher.
- `h5-reader/notes/` — design docs for you and for future sessions.

## What you don't own

- `src/` — the nmr_shielding library. Never read, never link.
- `ui/` — the single-conformation viewer. Read-only. Copy patterns,
  do not share code or targets.
- `fileformat/` — the H5 serialiser. Read-only. `analysis_file.cpp`
  compiled into this target via source include.
- `extern/HighFive/` — header-only HDF5 wrapper. Read-only.
- Top-level `CMakeLists.txt` — do not modify. The reader's build is
  self-contained under `h5-reader/`.

## The discipline (see memory: feedback_qt_discipline)

Before writing any Qt code in a session, invoke the `qt6-cpp` skill and
read the relevant `references/` file for the topic at hand.

- Every `QObject` constructor: `CENSUS_REGISTER(this)`.
- Every signal/slot connection: `ACONNECT(...)`, not raw `QObject::connect`.
- Every thread-sensitive method: `ASSERT_THREAD(this)` at the top.
- Structured logger installed in `main()` before anything else. Per-module
  `Q_LOGGING_CATEGORY` categories. Messages flow to stderr AND to UDP
  port 9997.
- UDP log is the primary debugging channel. When something misbehaves,
  `tail -f` the log stream BEFORE speculating about cause.
- No `QTimer` on interactive controls. `QtPlaybackController` is the one
  class whose timer is legitimate — timing is its responsibility.
- Full error handling at every external-library boundary (HighFive, any
  future QNetworkAccessManager etc.). Try/catch at the seam, convert to
  log line, continue with degraded state.
- Class-driven everywhere. `QFile`/`QDir`/`QStandardPaths` over POSIX.
  VTK pipeline (`vtkAlgorithm` → `vtkPolyData` → `vtkMapper` → `vtkActor`)
  over raw geometry construction. `QAbstractItemModel` for trees.

## Scope (see memory: project_h5_north_star)

Single protein, trajectory-animated. `QtConformation` is the trajectory;
`QtFrame` is one sampled XTC frame. The H5 is the authoritative data
backing. Rendering reflects per-frame H5 values; re-evaluation happens
only for volumetric BS/HM isosurfaces (which the H5 does not carry).

See `notes/SCOPE.md` for the full scope statement.

## Cross-platform bar (see memory: project_h5_reader_scope)

Builds on Windows MSVC, macOS Clang, Linux GCC. `CMakePresets.json`
defines per-platform presets. No platform-specific C++ outside
`#ifdef` gates in the diagnostics layer. All file I/O via Qt
abstractions. All networking via `QUdpSocket`/`QTcpServer`.

## Viewer hard-won lessons carried over (see memory: project_viewer_hardwon_lessons)

These patterns were won through three restart cycles of the existing
`ui/` viewer. Defaults here; divergences need a reason:

- CMake: `vtk_module_autoinit`, `CMAKE_AUTOMOC ON`, HDF5 include order
  fix (`BEFORE PRIVATE /usr/include/hdf5/serial` on Linux where 1.10
  is the link lib).
- Startup order: `QSurfaceFormat` before `QApplication`, `qRegisterMetaType`
  for every cross-thread payload, `aboutToQuit` → `shutdown()` for
  pre-GL-teardown cleanup.
- Shutdown: stop all timers, cancel workers, `renderWindow_->Finalize()`
  before Qt destroys GL context.
- Picking: ray cast + nearest atom with `devicePixelRatioF()` hi-DPI
  correction and Y-axis flip. NOT vtk pickers (they fail on imposter
  molecular rendering).
- External-library boundary: try/catch at the HighFive seam, log the
  exception with specific context, continue with partial/empty state.
- Log dock reads UDP; UDP port is consumed by exactly one socket on
  Linux — don't run `udp_listen.py` alongside the reader.

## Editor / language-server setup

`build/<preset>/compile_commands.json` is generated by CMake
(`CMAKE_EXPORT_COMPILE_COMMANDS ON`). Editors that use clangd should be
pointed at that directory via `--compile-commands-dir=build/<preset>`.
**Do not** symlink or copy the file to the project root — symlinks break
on Windows, and a stale copy drifts from the build reality. See README.md
for VS Code / Neovim / CLion specifics.

If you see clang diagnostics in your editor claiming "QObject file not
found" or "unknown type QString", clangd has not been pointed at the
build directory. That is an editor-configuration issue, not a code issue
— verify by running `cmake --build` and watching it succeed.

## What this directory does NOT do

- Animate using the library's `BiotSavartResult::SampleShieldingAt()` —
  we don't link the library. Volumetric BS/HM grids are re-evaluated
  by `QtBiotSavartCalc` / `QtHaighMallionCalc` classes in `src/calculators/`
  using closed-form math and the same TOML the extractor uses.
- Compute per-atom kernel values. Those are in the H5.
- Show prettified summaries that aren't in the object model
  (feedback_ui_data): UI fields match H5 dataset names and typed object
  properties.
