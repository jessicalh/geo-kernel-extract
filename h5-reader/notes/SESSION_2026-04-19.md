- **Shared-memory server.** `h5reader-server` long-running process
  loads the fleet into POSIX shared memory (`/dev/shm/h5reader/`),
  exposes a control socket, publishes a manifest. Each AI session
  `import h5reader` attaches zero-copy. Big data: no IPC. 128 GB RAM
  fits the 10-protein set comfortably.
- **Extension module only; embedded binary dropped.** An AI agent in
  a Bash shell restarts the process each prompt — embedded
  `h5reader --script foo.py` would pay 20 GB of load per
  interaction. `import h5reader` is the right shape. Qt binary stays
  for professors' GUI only.
- **Scope narrows to 10 proteins.** No 685. `fleet_calibration-
  working/` is this tool's universe. 685-fleet tooling is separate.

**None of this is built.** No `h5reader-server` code exists. No
`PYTHON_CLI_SPEC.md` exists. The decisions are recorded in memory
(`project_h5reader_python_cli`) and here; implementation is next.

### 3. Design principles (codified in forensics PRINCIPLES.md)

- **Huxley discipline.** Sit down before fact as a little child. No
  preconceived notions. Applied concretely: do not filter what the
  databases give; complain loudly about every inconsistency; no
  silent translations or aliases.
- **Lens, never filter.** Typed vocabulary is a lens over raw data,
  never a filter. Every typed path has a raw escape hatch. Enums
  expose `.raw_int`/`.raw_name`; predicates stringify + expose
  `.mask`; returned arrays are plain numpy with semantics via
  side-channel metadata; relationships are callable queries with
  explicit policy; dynamic field access always works; discovery is
  first-class.
- **Sidechain-named atoms are primary identity.** `(residue_type,
  atom_name)` tuple. Coarse groupings (AtomType, backbone, depth)
  are derived lenses, never primary.

## Key findings

- **1DV0/4757 resolved.** HHR23A UBA domain; BMRB residue numbering
  +2 offset from our PDB (construct numbering vs re-numbered
  1-based). `[residue_seq_offset] offset = -2` applied in both
  per-protein tables. Audit clean with 1 document-and-carry anomaly
  (N-term NH3+).
- **N-terminal NH3+ pattern.** Observed in 1DV0; expected at 1HS5.
  BMRB records a single backbone H at residue 1 where CHARMM has
  zwitterionic NH3+ (H1/H2/H3). Fundamental sample-vs-force-field
  mismatch; downstream policy.
- **RefDB re-referencing non-trivial for 4/10 proteins** (first-pass,
  to verify per protein): 1DV0 (125), 1HS5 (96), 1I8X (53), 1HD6
  (37). Downstream analysis must use RefDB `corrected_value` for
  these.
- **HSP protonation in 5/10 proteins** (first-pass): 1BWX, 1CBH,
  1G26, 1I2V, 1I8X. CHARMM doubly-protonated; BMRB silent on
  protonation. Requires deposition-metadata check per residue.
- **ILE `CD`/`CD1` naming divergence** (CHARMM no-locant, BMRB
  IUPAC locant). Encoded in starter; must carry to every per-
  protein table that involves ILE.

## Explicitly deferred

The audit does NOT decide — downstream policy per protein:

- Pseudo-atom expansion (methyl, NH3+, NH2).
- Stereo-ambiguity handling (ambig ≥ 2).
- Unmatched atom handling.
- RefDB-vs-BMRB preference.
- HIS protonation resolution.
- N-terminal NH3+ vs single-H (1DV0, 1HS5 pattern).
- 1DV0 offset polarity (if a future protein has +2 in the other
  direction).

## Cross-cutting discipline for the successor

A session coming to this cold should read `PRINCIPLES.md` before
anything else. The seven principles are the spec for how we think
about upstream data, not a convenience. Violations manifest as
false-matched rows, silent translations, and thesis-grade data
quality holes found too late.

Empiricism. No assumptions. Every protein is its own case.
Orthogonal records across databases. AI observations on every
anomaly. Sane resolution (adjudicate / escalate / defer-
downstream / document-and-carry). Recorded everywhere.
