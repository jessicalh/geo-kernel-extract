# papers/

Three paper-scale documents related to the thesis. Each gets its own subdirectory.

```
papers/
├── thesis/             — the thesis itself (not yet started)
├── annotated-bib/      — the 8-item annotated bibliography for programme submission
└── outline/            — the literature-review outline for programme submission
```

## Customary working pattern

Each paper is a text source (Markdown now, LaTeX eventually) with inline tags for figures, references, and cross-links. Graphs and layout assets live alongside the source. The build produces a PDF.

**Text → LaTeX → PDF** is the long-term pattern; for now, Markdown drafts live here and get promoted to LaTeX once the content is stable enough that layout matters. Figures get dropped into the same subdirectory as the text that references them.

## Future work

- **BibTeX integration with Zotero.** The `zotero` MCP server (see `~/src/zmcp/`) can fetch citation data; adapting its output to a `.bib` file under each paper subdirectory is deferred to a later session. For now, references are written out by hand in the Markdown drafts.
- **LaTeX scaffolding.** A shared preamble, thesis/article template, and a build script (likely `latexmk`-based) will land when the first paper moves from Markdown draft to LaTeX.
- **Figures.** Graph and diagram generation lives alongside the text that uses them; for the thesis, figures generated by the Python analysis pipeline in `../learn/` and `../analysis-speculative/` will be symlinked or copied into the relevant paper subdirectory.
