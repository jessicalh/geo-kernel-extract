# Topology substrate implementation plan — 2026-05-05

Implementation plan for the eight pending semantic fields in
`OBJECT_MODEL.md:2486-2498` plus six bonus fields surfaced by the
research pass. This is the central design doc for Phase 5 of
`session-handoff-20260503.md`. Companion: `spec/plan/topology-fields-research-2026-05-05.md`
(the research dossier; this doc operationalises it).

**Status**: design committed 2026-05-05 conversation. Implementation
begins after spec acceptance. Sequencing: B (this) → C+D (substrate
+ conformation companion CR landing) → E (IUPAC/BMRB projections).

---

## Architecture in one paragraph

`LegacyAmberTopology` carries 14 typed semantic fields per atom.
Values are populated at `Protein::FinalizeConstruction` via lookup
into a static C++ table compiled from the wwPDB Chemical Component
Dictionary plus literature. The table is generated by a build-time
C++ binary that links cifpp (CCD reader) and RDKit (CIP labeler,
canonical rank, aromatic + hybridization perception). The generator
reconciles four sources (cifpp/CCD, RDKit, Markley 1998, BMRB) under
explicit precedence rules and emits a checked-in
`src/generated/LegacyAmberSemanticTables.cpp` with constexpr typed-enum
arrays. The runtime library does not link RDKit; the only link to
chemistry-string libraries is cifpp's existing PDB-loading-boundary
use. Conformation-dependent geometry (peptide ω, ring-flip state, sp2
pyramidalization, ring-pucker phase) lands as a per-frame
`PlanarGeometryResult` ConformationResult companion, paired with the
substrate fields in the same H5 emission group.

## The string barrier — structural enforcement

The discipline rule: chemistry strings exist only at the loading
boundary; typed enums cross into substrate.

**Enforcement points**, in order of strength:

1. **Linker-level**: `libnmr_shielding.a` does not link RDKit. The
   generator binary `tools/topology/build_semantic_tables` is the
   only artifact in the build that links RDKit. Calling RDKit from
   runtime code is a link failure, not a code-review issue.

2. **Module-level**: gemmi / cifpp / RDKit `#include` directives are
   permitted in three locations and nowhere else:
   - `tools/topology/build_semantic_tables.cpp` (the generator,
     build-time tool, never linked into the runtime library).
   - `src/PdbFileReader.cpp` (existing — cifpp at the PDB-loading
     boundary; pre-existing sanctioned use per PATTERNS.md
     "cifpp and CCD: construction-time only").
   - `src/DsspResult.cpp` (existing — cifpp at DSSP construction).

   No new `#include "cifpp/..."` or `#include "rdkit/..."` lines
   land outside those files in this work or any subsequent work.

3. **Header-level**: zero headers under `src/` include cifpp or
   RDKit. The semantic table is emitted as a `.cpp` file that
   includes only `<array>`, `<cstdint>`, and our typed-enum
   headers.

4. **Field-level**: zero `std::string` fields appear in
   `LegacyAmberTopology`, in the `AtomSemanticEntry` table row
   struct, or in the `SemanticProvenance` typed record. Provenance
   carries enum source tags and packed enum value witnesses, not
   strings. Strings appear only when the H5 emission boundary
   renders typed enums to human-readable column labels.

5. **Test-level**: a `tests/test_string_barrier.cpp` runs at every
   ctest invocation and asserts:
   - `grep -l "rdkit\|RDKit\|GraphMol" src/*.{h,cpp}` → empty.
   - `grep -l "gemmi" src/*.{h,cpp}` → empty.
   - `grep -l "cif::\|cifpp" src/*.{h,cpp}` → exactly
     `PdbFileReader.cpp` and `DsspResult.cpp`.
   - The generated `src/generated/LegacyAmberSemanticTables.cpp`
     contains no `std::string` literals (only typed-enum values).

The five enforcement points are layered: each tightens the barrier
without depending on the others to hold.

**What this means in practice**: when a calculator needs to know
"is this atom a backbone amide H?", it reads
`tp.LegacyAmber().PolarHKindAt(atom_idx) == PolarHKind::BackboneAmide`.
It does not call cifpp, does not query gemmi, does not include
RDKit. The lookup is one cache-line read into the typed substrate.

## The 14 fields

Eight from `OBJECT_MODEL.md:2486-2498` plus six from the research
dossier. All are static, all derivable from `(AminoAcid, atom_local_idx,
protonation_variant_idx)`, all populated at `FinalizeConstruction`.

Per-field definitions, citations, and enum shapes are in
`spec/plan/topology-fields-research-2026-05-05.md`. This doc
records the runtime types as they appear in `LegacyAmberTopology.h`.

| Field | Type | Lines |
|---|---|---|
| Locant | `enum class Locant : uint8_t` | 7 values + None |
| BranchAddress | `struct { uint8_t outer; uint8_t inner; }` | two-level for Arg η |
| DiastereotopicIndex | `enum class : uint8_t` | None / Two / Three |
| ProchiralStereo | `enum class : uint8_t` | None / ProR / ProS / Unassigned |
| PlanarGroupKind | `enum class : uint8_t` | 8 values + None |
| PlanarStereo | `enum class : uint8_t` | None / E / Z / Unspecified |
| PseudoatomMembership | `struct { kind, locant, branch, in_super_group }` | M/Q/R per Markley Table 1 |
| PolarHKind | `enum class : uint8_t` | 12 values |
| RingPosition | `struct { primary, secondary }` of `RingMembership` | for fused rings |
| `aromatic` | `bool` | RDKit perception |
| Hybridization | `enum class : uint8_t` | sp / sp2 / sp3 / sp3d / sp3d2 / unset |
| BondOrderMask | packed per-neighbor bond order (single/double/aromatic/delo) | up to 4 neighbours |
| `formal_charge` | `int8_t` | -1 / 0 / +1 |
| `is_exchangeable` | `bool` | derived from PolarHKind |
| `equivalence_class` | `uint8_t` | RDKit canonical rank within residue |

Total per-atom storage: ~24 bytes (fits in a cache line). Per-residue
table: ~15 atoms × ~24 bytes = ~360 bytes. 20 standard residues × 5
variants average = ~36 KB total. Trivial.

Each enum value gets a docstring with chemistry citation in
`LegacyAmberTopology.h` per the standard set by `Ring::Intensity()`
and the existing typed-property pattern.

## Typed provenance shape

Per-atom-per-field, the substrate carries a compact typed witness of
which sources produced which value:

```cpp
// One byte per source-and-value pair. Up to 4 sources per field
// (cifpp/CCD, RDKit, Markley, BMRB).
struct SemanticSourceWitness {
    SemanticSource source;       // which source produced this value
    uint8_t        value_packed; // value cast to uint8_t (enum width)
};

struct SemanticProvenance {
    uint8_t                        primary_source_idx;  // index into witnesses
    SemanticConfidence             confidence;
    bool                           sources_agree;
    bool                           cip_verified;        // RDKit CIP ran and agreed
    std::array<SemanticSourceWitness, 4> witnesses;     // None entries are zeroed
};
```

Per-atom-per-field provenance: ~10 bytes. For 14 fields × ~15 atoms ×
20 residues × 5 variants ≈ 200 KB total table. Trivial.

**No strings.** All four enum types (`SemanticSource`,
`SemanticConfidence`, witness value, primary index) are typed. The H5
emission boundary renders enum names as human-readable strings only
at write time; runtime never round-trips through them.

`SemanticSource` enum:

```cpp
enum class SemanticSource : uint8_t {
    None                       = 0,
    cifpp_CCD_pdbx_aromatic    = 1,
    cifpp_CCD_pdbx_stereo      = 2,
    cifpp_CCD_value_order      = 3,
    cifpp_CCD_leaving_atom     = 4,
    cifpp_CCD_ordinal          = 5,
    RDKit_CIPLabeler           = 6,
    RDKit_CanonicalRank        = 7,
    RDKit_Hybridization        = 8,
    RDKit_AromaticPerception   = 9,
    RDKit_BondType             = 10,
    Markley1998_Table1         = 11,
    Markley1998_Figure1        = 12,
    IUPAC_1969                 = 13,
    BMRB_NomenclaturePage      = 14,
    BMRB_PseudoatomTable       = 15,
    AmberVariantTable          = 16,
    SynthesizedFromChemistry   = 17,
    AlternationDefault         = 18,  // CIP-untested fallback (should be empty after RDKit pass)
    ManualOverride             = 19,
};
```

`SemanticConfidence` enum:

```cpp
enum class SemanticConfidence : uint8_t {
    None                  = 0,
    AlgorithmAuthoritative = 1,  // single algorithmic source, no cross-check needed
    CIPVerified            = 2,  // RDKit CIP and Markley table agree
    SourcesAgree           = 3,  // multiple sources agree
    DisagreementLogged     = 4,  // sources disagree, primary chosen per precedence
    ManuallyOverridden     = 5,
};
```

## Reconciliation precedence — authoritative

Per-field source precedence. `Primary` wins when sources disagree;
`DisagreementLogged` confidence flag fires; all source values are
stashed in the witness array.

| Field | Primary | Cross-check 1 | Cross-check 2 | Disagreement → |
|---|---|---|---|---|
| Locant | mechanical from atom name | — | — | impossible |
| BranchAddress | mechanical from atom name | RDKit CanonicalRank | — | log + mechanical |
| DiastereotopicIndex | mechanical from atom name | — | — | impossible |
| ProchiralStereo | RDKit CIPLabeler | Markley Fig 1 | CCD pdbx_stereo_config | log + RDKit |
| PlanarGroupKind | Synthesized from chemistry | RDKit aromatic + bond perception | — | log + chemistry |
| PlanarStereo | CCD pdbx_stereo_config | Markley Fig 1 | — | log + CCD |
| PseudoatomMembership | Markley 1998 Table 1 | BMRB pseudoatom table | — | log + Markley |
| PolarHKind | Synthesized from chemistry | — | — | single source |
| RingPosition | Synthesized topology | RDKit SSSR | CCD ring perception | log + chemistry |
| `aromatic` | RDKit aromatic perception | CCD pdbx_aromatic_flag | — | log + RDKit |
| Hybridization | RDKit hybridization perception | — | — | single source |
| BondOrderMask | RDKit bond perception | CCD value_order | — | log + RDKit |
| `formal_charge` | AMBER variant table | RDKit | — | log + AMBER |
| `is_exchangeable` | derived from PolarHKind | — | — | derived |
| `equivalence_class` | RDKit CanonicalRank | — | — | single source |

**Methodological principle**: algorithmic sources (RDKit CIPLabeler,
RDKit CanonicalRank, etc.) win as primary because their methodology
is citable, version-pinned, and reproducible. The thesis methods
chapter cites "RDKit version 2023.09.6 CIPLabeler" as a defensible
source. Hand-curated tables and historical text are cross-checks
that we honour substantively (disagreements get logged for review),
but algorithmic primary stands as policy.

The synthesized fields (PlanarGroupKind, PolarHKind, RingPosition
labels) are cited per-enum-value in `LegacyAmberTopology.h` against
biochemistry literature, following the precedent of
`Ring::Intensity()` with its Case 1995 citation. These are
first-class chemistry contributions, not weaker provenance.

## Build pattern

**Runtime build (default)** — produces `libnmr_shielding.a` and
binaries:

1. `cmake -B build` — configures without the generator subproject.
2. `cmake --build build` — compiles `src/generated/LegacyAmberSemanticTables.cpp`
   alongside the rest of `src/`. No RDKit dependency.

**Regeneration build (opt-in)** — produces a fresh table file:

1. `cmake -B build-gen -DNMR_BUILD_TABLE_GENERATOR=ON`.
2. `cmake --build build-gen --target build_semantic_tables` —
   produces `tools/topology/build_semantic_tables` binary.
3. `./build-gen/tools/topology/build_semantic_tables \
       --ccd data/ccd/components.cif \
       --output src/generated/LegacyAmberSemanticTables.cpp` —
   reads CCD, runs RDKit, emits the C++ source.
4. Diff `src/generated/LegacyAmberSemanticTables.cpp` against git
   HEAD; commit if changes. Diff should be small or zero between
   regenerations unless inputs change.
5. Re-run normal `cmake --build build` to compile against the new
   table.

The generator is **only run when chemistry sources change**: a CCD
update from wwPDB, an RDKit version bump that changes CIP labeling,
or a new residue added to the canonical set.

## Test surface

### Test 1: string barrier discipline (CI)

`tests/test_string_barrier.cpp` runs grep-based assertions on the
source tree. Lightweight; runs in <1 second. Fails the suite if any
of the five enforcement points (above) is violated.

### Test 2: per-field chemistry expectations (per residue)

`tests/test_legacy_amber_topology_semantics.cpp` exercises each of
the 14 fields against the canonical residue chemistry. Examples:

```cpp
TEST(SemanticTables, BackboneAmideOnEveryStandardResidueExceptPro) {
    // Every standard 20 residue except Pro has a BackboneAmide H.
    for (auto aa : kStandard20) {
        if (aa == AminoAcid::PRO) continue;
        EXPECT_TRUE(HasAtomWithPolarHKind(aa, PolarHKind::BackboneAmide))
            << aa << " missing BackboneAmide N-H";
    }
    EXPECT_FALSE(HasAtomWithPolarHKind(AminoAcid::PRO, PolarHKind::BackboneAmide));
}

TEST(SemanticTables, ProchiralCipVerifiedForKnownPairs) {
    // The Markley Figure 1 worked example: Gly HA2 = ProR, HA3 = ProS.
    EXPECT_EQ(LookupProchiral(AminoAcid::GLY, "HA2"), ProchiralStereo::ProR);
    EXPECT_EQ(LookupProchiral(AminoAcid::GLY, "HA3"), ProchiralStereo::ProS);

    // β-onward methylenes follow the typical alternation: 3 = R, 2 = S.
    EXPECT_EQ(LookupProchiral(AminoAcid::ASP, "HB2"), ProchiralStereo::ProS);
    EXPECT_EQ(LookupProchiral(AminoAcid::ASP, "HB3"), ProchiralStereo::ProR);

    // Provenance for these MUST be CIPVerified.
    auto provenance = LookupProchiralProvenance(AminoAcid::GLY, "HA2");
    EXPECT_TRUE(provenance.cip_verified);
}

TEST(SemanticTables, RingPositionForPheBenzene) {
    // Phe ring atom positions per chemistry convention.
    EXPECT_EQ(LookupRingPosition(AminoAcid::PHE, "CG").primary.position,
              RingPositionLabel::Ipso);
    EXPECT_EQ(LookupRingPosition(AminoAcid::PHE, "CZ").primary.position,
              RingPositionLabel::Para);
}
```

Coverage requirement: every atom in every standard residue and every
variant has all 14 fields populated with non-`None` provenance source.

### Test 3: planar geometry CR (per frame)

`tests/test_planar_geometry_result.cpp` exercises the per-frame
planar-geometry calculator on 1ubq and the AMBER fleet fixtures.
Asserts ω angle within ±15° of canonical for non-Pro residues,
ring-flip state classification consistent with χ2 sign, sp2
pyramidalization within tolerance.

### Test 4: round-trip H5

The smoke test (already present) gains entries for the 14 substrate
fields and the planar-geometry CR outputs. Bless v2 tolerance applies.

### Test 5: regeneration determinism

`tests/test_table_regeneration.cpp` (only runs when generator binary
is available) re-runs the generator into a temp file and asserts it
matches the checked-in `src/generated/LegacyAmberSemanticTables.cpp`
byte-for-byte. Catches drift between checked-in tables and current
RDKit/cifpp.

## What this rules out

To be authoritative on what we are NOT doing:

- We are not linking RDKit into `libnmr_shielding`. RDKit lives only
  in the generator binary.
- We are not invoking RDKit at runtime, ever. The generator runs at
  build time only.
- We are not asking the user to adjudicate per-residue source
  disagreements. The reconciliation precedence table is policy;
  generator applies it; downstream stats reads the witness stash.
- We are not splitting the substrate landing across fields.
  All 14 fields land together in one substrate-extension session.
- We are not landing the substrate without its conformation
  companion CR. The two land paired in C+D.
- We are not deferring projections. Session E happens — the user
  identified it as required for analysis to continue.
- We are not letting CCD strings into the runtime substrate. Strings
  appear in the generator's output (typed-enum literals in C++
  source) and at the H5 emission boundary (column labels rendered
  for human consumption); nowhere else.
- We are not "carrying named debt". The CIP-verification debt
  dissolves into the RDKit-primary methodology — no manual
  per-residue chemistry verification step is required.
- We are not pre-emptively framing the synthesized fields
  (PlanarGroupKind, PolarHKind, RingPosition labels) as weaker
  contributions. Each enum value has a chemistry citation; the
  taxonomy is owned and defensible.

## Risks and mitigations

| Risk | Mitigation |
|---|---|
| RDKit CIP labeler produces a result that disagrees with Markley 1998 on a specific atom | Witness stash records all source values; test surface flags disagreements; methods chapter cites RDKit version |
| Generated table size makes compile slow | Per-residue tables are ~360 bytes; total ~36 KB; no compile-time concern |
| Build fails on machines without RDKit | Generator is opt-in (`NMR_BUILD_TABLE_GENERATOR=ON`); checked-in generated file means runtime build doesn't need RDKit |
| Future calculator wants to know ring-position chemistry but goes back to string lookup | `LegacyAmberTopology` accessors named to be the obvious choice; PATTERNS.md §19 documents the substrate as the preferred path |
| CCD update changes a field for an existing residue | `tests/test_table_regeneration.cpp` flags drift; user reviews and decides whether to accept new tables |
| Variant-protonation interactions miss a case | Coverage test asserts every (residue, variant) pair has every field non-`None`; the "every atom in every variant" assertion is comprehensive |
