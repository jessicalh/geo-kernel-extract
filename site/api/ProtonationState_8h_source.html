2//
3// ProtonationState: per-residue protonation decisions for a protein.
4//
5// This is load-bearing, not metadata:
6// - Determines partial charges (ASP=-1, ASH=0)
7// - Determines ring type (HID vs HIE vs HIP → different RingTypeIndex)
8// - Determines which atoms exist (ASH has one more H than ASP)
9// - Is the input to tleap (ORCA prep) and pdb2gmx (GROMACS prep)
10//
11// A ProtonationState is a VALUE TYPE. It is created by a protonation tool
12// (PROPKA, KaML, tleap default) and consumed by a topology builder (tleap,
13// pdb2gmx) to produce a Protein with the correct hydrogen atoms and charges.
14//
15// The variant index into AminoAcidType::variants is the canonical form.
16// The NamingRegistry translates to tool-specific names at the tool boundary.
17//
