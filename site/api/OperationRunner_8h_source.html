1#pragma once
2//
3// OperationRunner: the ONE home for all ordered sequences.
4//
5// THIS IS ONLY FOR ORDER. The conformation is the buffer where
6// results accumulate. The runner does not hold state, does not
7// cache intermediate results, does not decide what to compute.
8// It sequences operations. If a step's prerequisites are on the
9// conformation, the step runs. If not, it is skipped. Nothing
10// runs backwards. Nothing runs twice. These are the ONLY
11// variations — the presence or absence of upstream results.
12//
13// Every use case (UI, CLI, batch, training) calls one of
14// these methods. The conformation arrives with its protein
15// identity determined (protonated, topology resolved). The
16// runner fills it with computed results in dependency order.
17//
18// All ordered sequences live here. Future agents add new
19// sequences here, not in new files.
20//
21// Replaces both Pipeline.h and CalculationRunner.h.
22//
